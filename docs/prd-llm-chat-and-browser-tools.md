# 产品需求文档（PRD）：LLM Chat 与浏览器交互

> 产品: Issue Manager (VS Code Extension + Chrome Extension)
> 版本: 3.0.115 | 日期: 2026-03-09

## 1. 产品背景

Issue Manager 是一个以 issueMarkdown 为核心的知识管理工具，通过 VS Code 扩展提供笔记的创建、编辑和树形组织能力，通过 Chrome 扩展提供网页内容采集能力。

随着 LLM 能力的普及，用户期望在日常知识管理流程中直接与 AI 交互，而非手动执行搜索、整理、记录等重复性工作。同时，Chrome 浏览器作为主要的信息获取入口，用户希望 AI 能直接操控浏览器完成页面交互任务。

## 2. 目标用户

- **知识工作者**: 日常需要大量搜索、阅读、整理信息的研究人员、产品经理、工程师
- **笔记重度用户**: 使用 Markdown 笔记系统管理个人知识库的用户
- **浏览器效率用户**: 希望自动化浏览器操作（批量登录、表单填写、信息采集）的用户

## 3. 核心需求

### P0 — 必须有

#### 3.1 LLM 多会话对话
- **需求**: 用户可在 Chrome 侧面板/VS Code 中与 LLM 进行多轮对话
- **验收标准**:
  - 支持创建、切换、删除对话
  - 对话记录持久化，刷新不丢失
  - 支持流式输出（打字机效果）
  - 对话历史作为上下文传入 LLM

#### 3.2 LLM 工具调用（Tool Calling）
- **需求**: LLM 可根据用户意图自动调用预定义工具
- **验收标准**:
  - 模型能判断何时需要调用工具
  - 工具执行结果回传 LLM 继续对话
  - 支持多轮工具调用（最多 10 轮）
  - 用户可看到工具调用过程反馈

#### 3.3 笔记管理工具
- **需求**: LLM 可创建、搜索、读取、更新 issueMarkdown 笔记
- **验收标准**:
  - `create_issue_tree` 可一次创建多层级笔记
  - `search_issues` 支持模糊搜索
  - 创建的笔记在 VS Code 文件树中立即可见

### P1 — 应该有

#### 3.6 工具调用 UI 反馈
- **需求**: 用户能实时看到工具执行状态
- **验收标准**:
  - 调用中显示旋转动画和工具名
  - 完成后显示成功/失败状态
  - 可点击展开查看执行结果详情
  - 失败时红色高亮提示

#### 3.7 Web Research Agent
- **需求**: 一键启动自主研究任务，Agent 自动搜索→抓取→整理→记录
- **验收标准**:
  - 输入任务描述即可启动
  - 实时展示进度（搜索了什么、访问了什么、创建了什么）
  - 研究结果自动保存为层级笔记
  - 支持取消正在运行的任务

### P2 — 可以有

#### 3.8 Chrome 面板对话持久化
- **需求**: Chrome 面板的对话记录持久化到 issueMarkdown 文件
- **验收标准**:
  - 对话列表从 VS Code 后端加载
  - 新消息实时追加到文件
  - 支持跨设备 Git 同步

#### 3.9 模型选择
- **需求**: 用户可选择不同的 Copilot 模型
- **验收标准**:
  - UI 展示可用模型列表
  - 记忆用户上次选择
  - 自动回退（指定模型不可用时回退到 gpt-4o → gpt-4.1 → 任意）

#### 3.10 角色系统
- **需求**: 支持预设不同系统提示词角色
- **验收标准**:
  - 内置多种角色（研究助手、翻译、代码审查等）
  - 可切换角色，切换后影响后续对话
  - 角色配置可自定义

## 4. 非功能需求

### 4.1 性能
- 工具执行超时: 10-20 秒
- 页面元素获取限制: 最多 80 个
- 工具结果截断: 300 字符 preview
- 流式响应延迟: < 500ms（首 token）

### 4.2 安全
- CSS 选择器使用 `CSS.escape()` 防注入
- 不执行任意 JavaScript（工具行为预定义）
- 工具仅在 Chrome 扩展授权范围内操作
- 敏感信息（密码）由用户主动提供，不自动推断

### 4.3 兼容性
- VS Code: 1.85+
- Chrome: 120+
- 依赖: GitHub Copilot 扩展
- 模型: 需支持 VS Code LanguageModel API 的 tool-calling
