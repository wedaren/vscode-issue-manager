# 深度调研功能使用说明

## 功能概述

深度调研功能是一个强大的 AI 辅助调研工具，可以帮助你针对特定主题进行深入的调研分析。它利用 LLM 的能力，结合本地笔记库的内容，生成全面的调研报告。

## 主要特性

### 三种调研模式

1. **自动模式 (Auto)**
   - 命令：`Issue Manager: 深度调研问题`
   - 自动选择最佳调研方式
   - 默认使用本地笔记模式

2. **本地笔记模式 (Local)**
   - 命令：`Issue Manager: 深度调研（本地笔记）`
   - 基于你的本地笔记库进行调研
   - 会引用相关的本地笔记
   - 适合需要结合现有知识的调研

3. **纯 LLM 模式 (LLM Only)**
   - 命令：`Issue Manager: 深度调研（纯 LLM）`
   - 仅使用 LLM 的知识进行调研
   - 不参考本地笔记
   - 适合全新主题或通用知识的调研

## 使用方式

### 方式一：通过命令面板

1. 按 `Cmd/Ctrl + Shift + P` 打开命令面板
2. 输入 `深度调研` 选择对应的命令
3. 在弹出的输入框中输入调研主题
4. 等待调研完成，文档将自动打开

### 方式二：通过深度调研视图

1. 在侧边栏找到"深度调研"视图
2. 点击工具栏的 `+` 按钮
3. 选择调研模式（本地笔记或纯 LLM）
4. 输入调研主题
5. 在视图中查看任务进度

## 视图功能

深度调研视图显示：

- **活动任务**：正在进行的调研任务
  - 显示任务状态（等待中、调研中、已完成、失败、已取消）
  - 可以取消正在运行的任务
  - 任务完成后自动打开生成的文档

- **历史文档**：已生成的调研文档
  - 按时间顺序显示
  - 点击文档直接打开
  - 可以删除不需要的文档

## 生成的文档结构

调研完成后，会在问题目录下生成一个 Markdown 文档，包含：

- **文档头部**：调研主题、模式、生成时间
- **调研概述**：主题的总体介绍
- **核心发现**：关键信息和要点
- **相关笔记引用**（本地模式）：参考的本地笔记列表
- **深度分析**：详细的分析内容
- **结论与建议**：总结和实用建议

## 技巧与建议

1. **明确调研主题**
   - 使用清晰、具体的主题描述
   - 例如："如何优化 TypeScript 项目的编译性能"

2. **选择合适的模式**
   - 有相关笔记时选择"本地笔记"模式
   - 新主题或通用问题选择"纯 LLM"模式

3. **管理调研文档**
   - 定期整理和归档调研文档
   - 为重要调研添加到问题树中

4. **取消任务**
   - 如果调研主题输入错误，可以快速取消任务
   - 点击任务旁边的停止按钮即可

## 注意事项

- 需要配置有效的问题目录才能使用此功能
- 需要安装并登录 GitHub Copilot 扩展
- 调研过程可能需要一些时间，请耐心等待
- 生成的文档保存在问题目录的根目录下

## 示例场景

### 场景 1：技术调研

**主题**：微前端架构最佳实践

**模式**：本地笔记（如果你已有相关笔记）

**结果**：综合你的笔记和 LLM 知识，生成全面的技术方案对比文档

### 场景 2：新知识学习

**主题**：Rust 语言的所有权系统

**模式**：纯 LLM

**结果**：基于 LLM 的知识库，生成系统的学习资料

### 场景 3：问题解决

**主题**：如何解决 VS Code 扩展的内存泄漏问题

**模式**：本地笔记

**结果**：结合你过往的经验笔记和最佳实践，提供解决方案

## 快捷键

目前没有默认快捷键，你可以在 VS Code 的键盘快捷方式设置中为以下命令配置：

- `issueManager.deepResearchIssue`
- `issueManager.deepResearchIssueLocal`
- `issueManager.deepResearchIssueLlmOnly`
